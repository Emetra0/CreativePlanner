# Translation Audit

## Goal

Move all user-visible app text to source-level translation using `useAppTranslation()` and localized date/time formatting, with source-controlled locale resources as the primary translation source and a DOM fallback for unmanaged static UI text.

## Method

1. Audit every `src/app/**/page.tsx` route.
2. Audit shared `src/components/**/*.tsx` surfaces that render visible UI.
3. For each file, check:
   - hardcoded English labels, buttons, headings, placeholders, tooltips, dialog copy
   - English-only date and time formatting
   - native alerts or confirms that bypass translated app dialogs
   - missing `useAppTranslation()` integration on visible UI surfaces
4. Keep source-level translation as the first choice for route/component-owned UI.
5. Keep built-in locale resources in `src/locales/**` for full-message translations and glossary terms.
6. Use the exact-text DOM fallback translation for static text nodes and attributes that are not yet wired at source level.
7. Patch one batch at a time.
8. Run a focused validation after each batch.
9. Keep this file updated with edits and verification.

## Status Key

- `pending`: not audited yet
- `in-progress`: currently being patched
- `partial`: audited, but more untranslated copy remains
- `done`: audited and patched for visible strings in scope
